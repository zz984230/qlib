## 核心概念修正

### 1. 风险控制概念

**错误理解**: 整体回撤限制3%
**正确理解**: 单笔交易止损2%账户值

海龟交易法则的风险控制核心：
- 每笔交易最大亏损 = 账户的2%
- 止损价格 = 入场价 - 2*ATR（约等于2%账户值）
- 整体回撤没有硬性限制，由市场表现决定

### 2. 动态策略选择

不是单一策略适应所有市场，而是根据市场状态切换策略：

| 市场状态 | 适用策略 | 预期特征 |
|---------|---------|----------|
| 强趋势 (ADX>30) | 海龟趋势跟踪 | 追逐趋势，让利润奔跑 |
| 弱趋势 (20<ADX<=30) | 轻仓海龟 | 降低仓位，谨慎跟随 |
| 震荡 (ADX<=20) | 均值回归策略 | 低买高卖，区间交易 |
| 异常波动 | 空仓观望 | 避免交易风险 |

## 策略实现

### 海龟趋势跟踪策略

**市场状态**: strong_trend, weak_trend

**核心规则**:
- 入场: 突破20日高点（唐奇安通道上轨）
- 止损: 入场价 - 2*ATR（约2%账户值）
- 移动止损: 2*ATR低于最高价
- 金字塔加仓: 每0.5*ATR加仓，最多4单位

**参数范围**:
- 信号阈值: 0.05-0.15
- 出场阈值: 0.05-0.15
- 止损ATR: 2.0-2.5
- 仓位: 强趋势100%，弱趋势60%

### 均值回归策略

**市场状态**: ranging

**核心规则**:
- 入场: RSI超买超卖、布林带触及、均值回归信号
- 止损: 1.5*ATR
- 目标: 回归到均值（如MA20）

**参数范围**:
- 信号阈值: 0.15-0.30
- 出场阈值: 0.10-0.20
- 止损ATR: 1.0-1.5
- 仓位: 25%

## 风险控制设计

### 单笔止损2%

```python
def calculate_unit_size(account_value: float, atr: float, price: float) -> int:
    """计算海龟单位大小

    规则：1单位 = 账户 * 1% / ATR
    止损 = 2*ATR，最大亏损 = 2%账户
    """
    risk_amount = account_value * 0.01  # 1%账户值
    unit_value = risk_amount / atr
    unit_shares = int(unit_value / price)

    # A股100股整数倍
    unit_shares = (unit_shares // 100) * 100

    return max(0, unit_shares)
```

### 整体回撤控制

不设置硬性整体回撤限制，但通过以下方式控制：
- 市场状态过滤：异常波动市空仓
- 趋势强度过滤：弱趋势降低仓位
- 单笔止损：严格止损限制单笔亏损

## 有效策略判定

修正后的判定标准：

1. **单笔止损合规**: 所有交易止损<=2%账户值
2. **夏普比率**: 1年期夏普比率>0.5
3. **胜率**: 至少40%胜率
4. **交易频率**: 年度交易20-100笔

移除"整体回撤<=3%"的硬约束。
